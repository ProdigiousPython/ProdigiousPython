10. String representations of objects: str() vs repr()Â¶

str() and repr() are builtin functions used to represent the object in the form of string.

Suppose we have an object x.

str(x) would be calling the dunder (double underscore) __str__ method of x as x.__str__()

repr(x) would be calling the dunder (double underscore) __repr__ method of x as x.__repr__()

ğŸ˜‘ Well, what all are these new terms __str__ and __repr__ ğŸ¤”?

As we know that Python is object oriented language, and so supports inheritance. In Python, all the classes would inherit from the base class object. object class has the methods __str__, __repr__ and a lot more (which can be deepdived in someother notebook ğŸ˜‰).
